# 移动互联网产品行业画像

> 适用于移动端应用产品的 PMF 验证场景：社交、工具、内容、游戏、本地生活、健康、教育、金融等各类 App 及微信小程序。
> 核心逻辑为"注意力极度稀缺，留存是生死线"，头部集中度极高，绝大多数产品在 30 天内被用户遗忘。

## 1. 行业概述

中国移动互联网用户规模约 10.8 亿，人均日使用时长超过 6.5 小时。但这个看似庞大的市场背后是一个极度残酷的竞争格局——用户手机上平均安装 80+ 个 App，但日均实际使用不到 10 个。这意味着绝大多数 App 处于"装了但不用"的僵尸状态。

留存数据揭示了移动互联网的生死线：App 平均 D30（30 天）留存率低于 5%。也就是说，100 个新用户中只有不到 5 个会在一个月后仍然打开你的应用。日活（DAU）超过 10 万的 App 在整个市场中占比不到 1%——这是绝大多数创业者不愿面对但必须认清的现实。

获客成本持续攀升是该行业的另一个结构性挑战。早期移动互联网的获客红利已经消失，社交裂变的效率在经过多年透支后显著下降。用户对"分享得福利""拼团砍价"等裂变机制产生了强烈的免疫甚至反感。在注意力极度碎片化的环境中，新产品面临的不是"能不能获得下载"的问题，而是"能不能在首次打开后的 5 分钟内说服用户留下来"的问题。

## 2. 用户决策模型

### 认知阶段

移动 App 的认知路径高度碎片化：**社交传播**（朋友推荐/群聊分享）、**内容平台种草**（在小红书/抖音看到推荐）、**应用商店发现**（搜索/榜单/编辑推荐）、**预装/系统入口**（手机厂商预装或系统推送）。认知阶段的时间窗口极短——用户从"看到"到"决定是否下载"通常在 3-10 秒内完成。如果产品无法在这个窗口内传递清晰的价值主张，机会就永远错过了。对新产品来说，"一句话说清楚你是什么"是认知阶段最核心的挑战。

### 评估阶段

与 3C 或 SaaS 不同，移动 App 的"评估"极其轻量且与"使用"重合——用户不会做详细的功能对比，而是直接下载试用。评估在首次打开后的 1-5 分钟内完成。这段时间内，用户形成三个判断：（1）"这是什么"——能否立即理解产品的核心功能；（2）"跟我有关吗"——核心价值与自身需求的匹配度；（3）"比我现在用的好吗"——与现有替代方案（包括"不用任何工具"）的对比。如果这三个判断中任何一个为否，用户退出且大概率不会再打开。

### 决策阶段

移动 App 的"决策"不是一次性事件，而是一个持续的留存过程。用户每天都在用行为投票："今天要不要打开这个 App？"驱动持续使用的核心力量分为三类：**习惯回路**（固定时间/场景触发使用）、**内容/社交牵引**（有新内容要看、有朋友在用）、**功能刚需**（必须用它完成某个任务）。没有以上任何一种力量的产品，会在新鲜感消退后迅速被遗忘。促销/补贴带来的活跃是虚假决策信号——用户在奖励消失后会立即离开。

### 复访/付费阶段

移动 App 的商业化路径通常是"先留存后变现"。免费用户的付费转化率因品类差异极大（工具类 2-5%，游戏类 3-8%，社交类 <2%）。付费意愿与使用深度强相关——只有深度使用者才会考虑付费，而深度使用者在总用户中占比通常不超过 15%。订阅制在中国移动用户中的接受度正在提升但仍远低于美国，"买断制"和"免费+广告"仍是主流预期。

## 3. PMF 信号矩阵

### 强 PMF 信号

- **D30 留存率 > 15%**——远超行业均值（<5%），说明产品解决了持续存在的真实需求
- **用户自发分享给朋友（无激励）**——不靠裂变机制的有机传播是最强 PMF 信号
- **卸载后重新安装**——用户离开后又回来，证明产品在用户心智中留下了不可替代的印记
- **使用频率自然上升**——用户从周活变为日活，使用深度自然增加
- **用户主动在社交媒体推荐**——自发产出使用体验内容（UGC），产品成为社交话题
- **自然搜索品牌词持续增长**——从品类搜索转向品牌搜索，品牌认知建立

### 弱 PMF 信号 / 伪 PMF 警告

- 刷榜/ASO 优化带来的下载量激增（虚假繁荣，下载数是最具欺骗性的虚荣指标）
- 补贴/裂变带来的用户增长（补贴停止后用户立即流失，"薅羊毛"用户无长期价值）
- 推送通知拉回的日活数据（被动打开≠主动使用，关掉推送后活跃度暴跌）
- 社交裂变带来的病毒系数（很多是"帮忙砍一刀"的人情传播，不代表产品价值认可）
- 短期内的高增长曲线（可能是营销驱动的脉冲，不是有机增长的稳态趋势）
- 应用商店高评分但低留存（可能是刷评或首次体验不错但缺乏持续价值）

## 4. OmniscientAgent 指引

### INIT 阶段行业特殊性

OmniscientAgent 在处理移动 App 产品时，必须在 INIT 阶段识别以下关键维度：

1. **首次体验设计**：产品能否在 5 分钟内让新用户体验到核心价值（Aha Moment）？这直接决定 D1 留存，而 D1 留存决定了后续所有指标的天花板
2. **使用频率假设**：产品的目标使用频率是日活级（社交/内容）、周活级（工具/效率）还是月活级（低频刚需）？不同频率的 PMF 信号评估标准完全不同
3. **网络效应类型**：产品有无网络效应？是同侧网络效应（用户越多越好用）还是跨侧网络效应（供给侧和需求侧互相促进）？网络效应的存在与否决定了增长曲线的形状
4. **冷启动策略**：产品如何解决"没有用户→没有价值→没有用户"的冷启动悖论？冷启动失败是移动 App 死亡的最常见原因

### Agent 配置建议

- Star Agent 应包含至少 1 个"科技/App 推荐类 KOL"和 1 个"目标品类垂直 KOL"
- 对社交类产品，Star Agent 需模拟"没有朋友在用"的冷启动场景——产品在社交空白状态下的吸引力
- OmniscientAgent 在评估留存信号时，必须用行业基准（D30 < 5%）作为锚点，而非模拟中 Agent 的乐观表态

## 5. Star/Sea 行为锚点

### Star Agent 行为特征

移动 App 行业的 Star Agent 分为两类：

**App 推荐类/科技博主**：以发掘和推荐新 App 为内容定位。这类 KOL 对新产品的初始态度相对开放——他们的内容模式就是"试用新 App 并给出评价"。但他们的评价标准极为苛刻：首次打开体验是否流畅、核心功能是否一目了然、与同类产品相比有何差异。如果产品在首次体验中没有亮点，他们的内容结论大概率是"鸡肋，不推荐"。他们的推荐对早期采纳者群体有直接影响力，但对大众用户的影响有限。

**垂直领域 KOL**：非科技类但在特定领域有影响力的 KOL（如健身博主推荐运动 App、理财博主推荐记账 App）。这类 KOL 的推荐更具场景说服力——"我实际用它做XX，效果是XX"。他们的受众更贴近大众用户画像，推荐转化率通常高于科技博主，但前提是产品与他们的内容领域自然契合。

### Sea Agent 行为分层

Sea Agent 按移动 App 采纳行为分为三个梯队：

| 梯队 | 占比 | 行为特征 |
|------|------|----------|
| 早期采纳 | ~10% | 喜欢尝试新 App，会在朋友间分享发现，对新产品容忍度高但要求"有意思" |
| 跟随下载 | ~20% | 看到多个信息源推荐后才会下载，首次体验不佳会立即卸载，需要明确的使用理由 |
| 不下载/下了不用 | ~70% | 对新 App 极度冷漠，手机上已有的 App 已满足需求，"又要下个新的？算了" |

**关键警告**：70% 的用户对新 App 的默认态度是"不下载"或"下了不用"。这不是因为产品不好，而是因为用户的注意力和手机空间都已被现有 App 占满。OmniscientAgent 在传播模拟中必须将这个基数作为硬性约束——即使产品优秀，穿透 70% 冷漠用户群的难度也远超想象。

## 6. 专属评分维度

> 以下维度供 OmniscientAgent 在移动互联网产品评估时选用，作为通用评分卡（5 个默认维度）的补充。

### 首次体验转化率（First-Session Conversion）

方向：1 = 首次即流失，5 = 首次即 Aha Moment

| 分数 | 行为锚点 |
|------|----------|
| 1 | 首次使用无法体验核心价值，用户打开后困惑或失望。"这个 App 到底是干嘛的？"——关掉，卸载。 |
| 2 | 需多次使用才能感受价值，首次体验不够直观。用户打开后"感觉有点意思但不知道怎么用"，大概率不会再打开。 |
| 3 | 首次使用能初步感知价值但需引导和学习。用户在引导流程完成后能理解核心功能，但需要投入时间学习才能真正上手。 |
| 4 | 5 分钟内能体验核心功能并感受到价值。用户在首次使用后会想"这个不错，明天再试试"。 |
| 5 | 首次打开即产生 Aha Moment，用户在 1-2 分钟内就说"哇，这个好用"。首次体验本身就是传播素材。 |

### 留存曲线健康度（Retention Curve Health）

方向：1 = 断崖下跌，5 = 快速趋平形成习惯

| 分数 | 行为锚点 |
|------|----------|
| 1 | D1 后断崖下跌，D7 留存 <5%。用户在首次体验后几乎全部流失，产品无留存基础。 |
| 2 | 持续下滑无趋平迹象，D30 <5%。用户缓慢但持续流失，没有形成任何使用习惯。与行业平均水平一致，但这本身就是"失败的常态"。 |
| 3 | 下滑后轻微趋平，D30 稳态 5-15%。产品对一小部分用户有持续价值，但多数用户仍会流失。有改进空间但 PMF 尚未达成。 |
| 4 | 明显趋平，D30 稳态 15-30%。留存曲线在 D7 左右出现明显拐点并趋于平稳。产品已建立一定的使用习惯，属于行业优秀水平。 |
| 5 | 快速趋平，D30 >30%，形成使用习惯。留存曲线在 D3-D5 即趋平，说明产品快速嵌入用户日常。这是强 PMF 的硬指标。 |

### 网络效应强度（Network Effect Strength）

方向：1 = 纯工具无网络效应，5 = 强网络效应赢家通吃

| 分数 | 行为锚点 |
|------|----------|
| 1 | 纯工具型，用户数不影响产品价值。每个用户独立使用，增长靠持续获客而非网络效应飞轮。 |
| 2 | 微弱网络效应，不构成增长飞轮。用户数增加对体验有轻微正向影响，但不足以驱动自增长。 |
| 3 | 存在网络效应但需达到临界量才能激活。冷启动阶段产品价值受限，一旦突破临界量会有显著加速。关键问题是"能否撑到临界量"。 |
| 4 | 明显网络效应，用户增长带动产品价值提升。已过临界量或接近临界量，用户增长和产品价值形成正循环。 |
| 5 | 强网络效应，赢家通吃潜力。每新增一个用户都显著提升全网价值，竞争对手在用户规模上的差距很难弥补。 |

## 7. 常见陷阱与反模式

### 评估错误

- **下载量幻觉**：下载量是移动互联网最具欺骗性的虚荣指标。D30 留存 <5% 是行业常态，100 万次下载可能只留下不到 5 万活跃用户
- **补贴裂变陷阱**：通过补贴/红包驱动的裂变增长与有机增长有本质区别。补贴停止后，用户留存率通常断崖下跌 60-80%，所谓的"病毒系数"瞬间归零
- **日活稀有性盲区**：日活 >10 万的 App 占比 <1%。创业者对"做到百万日活"的期望在统计意义上等同于"中彩票"
- **冷启动死亡螺旋**：社交/社区类产品在早期因用户少→内容少→体验差→用户更少而进入死亡螺旋。"产品本身不错但时机不对"是最常见的自我安慰

### LLM 乐观陷阱

- LLM 容易高估"好产品自然会被发现"的概率——在模拟中，Agent 可能假设优秀的产品会通过口碑自然传播，忽略了注意力市场的极度拥挤和用户对新 App 的默认冷漠
- LLM 容易低估留存的难度——D30 <5% 是常态而非例外。模拟中 Agent 表示"会继续使用"不等于真实用户行为，真实留存率应以行业基准为锚而非 Agent 表态
- LLM 容易混淆"首次体验好评"和"长期留存"——首次体验的新鲜感和持续使用的习惯养成是完全不同的机制。模拟中如果 Agent 对首次体验给出正面评价，不能直接推断留存
- LLM 容易高估网络效应的自然形成——在模拟中，Agent 可能假设"只要用户够多网络效应就会启动"，忽略了绝大多数产品根本到不了网络效应的临界量
