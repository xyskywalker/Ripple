# SaaS/软件服务行业画像

> 适用于企业级 SaaS 及软件服务产品的 PMF 验证场景：协作工具、CRM、ERP、HRM、数据分析平台、营销自动化、开发者工具、行业垂直 SaaS 等。
> 核心逻辑为"长周期决策、低付费意愿、高定制需求"，中国市场的 SaaS 渗透率远低于美国，PMF 验证需穿透组织决策链的层层阻力。

## 1. 行业概述

中国企业级 SaaS 市场规模约 800-1000 亿元，渗透率远低于美国市场。这个差距不仅是"发展阶段"的问题，更是付费文化和组织决策模式的结构性差异。中国企业对"持续为软件付费"的接受度显著低于美国，大量企业仍倾向于一次性买断部署或要求深度定制化，这从根本上冲击了 SaaS 的商业模式基础。

SaaS 行业的关键健康指标是年流失率（Annual Churn Rate），行业中位数约 5-7%。但这个数字掩盖了巨大的方差——头部 SaaS 产品年流失率可低至 2-3%，而早期产品或 PMF 未达的产品年流失率可达 20-30%。PLG（Product-Led Growth）模式下，免费→付费的转化率通常仅 2-5%，远低于北美 10-15% 的水平。NRR（Net Revenue Retention，净收入留存率）>100% 是被广泛认可的强 PMF 标志——意味着即使不新增客户，现有客户的扩展收入也能覆盖流失。

SaaS 创业最隐蔽的陷阱是"项目化"——客户的定制化需求可能将标准化产品拖入逐客户定制的泥潭，产品变成项目，团队变成外包。

## 2. 用户决策模型

### 认知阶段

企业 SaaS 的认知触发通常来自三种路径：**业务痛点驱动**（现有流程效率低、人力成本高）、**行业趋势推动**（同行开始使用、行业会议推介）和**上层指令**（管理层要求数字化转型）。认知阶段的核心特征是"需求模糊"——企业知道"有问题"，但不确定"软件能不能解决"以及"值不值得花这个钱"。对多数中国企业来说，软件采购的心理门槛远高于其他采购——因为软件失败的案例太多，决策者天然持怀疑态度。

### 评估阶段

评估是 SaaS 购买决策中最漫长、最痛苦的阶段。典型评估路径为：初步了解 → 产品演示 → 免费试用/POC → 内部评审 → 与 IT 部门对接 → 安全合规审查 → 预算审批。这条链路上的每个环节都可能卡住，周期从数周到数月不等。评估阶段的核心困境是 ROI 可证性——"用了你的产品能省多少钱/多赚多少钱"是采购决策者必须回答的问题，而大多数 SaaS 产品很难在试用阶段就给出清晰的量化答案。

### 决策阶段

SaaS 的采购决策链涉及多个角色：使用者（日常操作的员工）、部门负责人（预算审批）、IT 部门（技术评审和安全合规）、采购部门（商务谈判）。任何一个角色的否决都可能终止交易。在中国企业中，决策权通常高度集中——很多情况下最终决策取决于"一把手"的态度，而一把手关注的维度（战略价值、面子工程、关系因素）与使用者关注的维度（好不好用）可能完全不同。

### 续费/扩展阶段

首年续费是 SaaS PMF 最严苛的验证节点。大量 SaaS 产品能"卖出去"但"留不住"——首年签约可能靠关系、靠折扣、靠承诺，但续费完全取决于实际使用价值。续费时，使用者的声音权重显著上升："好不好用"压过"战略价值"成为核心标准。如果产品在日常使用中沦为"不得不用但很烦"的存在，续约谈判会变得极为艰难。

## 3. PMF 信号矩阵

### 强 PMF 信号

- **NRR > 100%**——现有客户自发增购，不靠新客也能增长收入，是 SaaS PMF 的黄金指标
- **使用者主动在组织内推广**——"冠军用户"自发说服其他部门使用，无需销售介入
- **POC 结束后客户拒绝停用**——试用期到了，使用部门要求立即签约，不愿回退到旧流程
- **客户主动推荐同行企业**——无需推荐激励，自发向同行推荐产品
- **续费率 > 90%**——高续费率是最硬的长期 PMF 证据
- **客户需求与产品路线图自然吻合**——客户想要的功能恰好在你的规划中

### 弱 PMF 信号 / 伪 PMF 警告

- 免费试用用户量大但付费转化极低（PLG 转化率 2-5% 是常态，大量试用≠付费意愿）
- 客户表示"感兴趣"但不推进到 POC（礼貌性了解占比 >70%，"感兴趣"≠会购买）
- 靠创始人个人关系拿下的大客户订单（无法复制的销售模式不是 PMF）
- 签约后实际使用率极低（"买了不用"是常见现象，可能是政绩采购或关系签单）
- 大量定制化需求才能完成交付（产品沦为项目，标准化 SaaS 变成外包公司）
- 只在深度折扣下才能成交（价格敏感≠产品价值认可）

## 4. OmniscientAgent 指引

### INIT 阶段行业特殊性

OmniscientAgent 在处理 SaaS 产品时，必须在 INIT 阶段识别以下关键维度：

1. **PLG vs SLG 路径**：产品是通过产品体验自底向上扩散（PLG），还是通过销售团队自上而下推进（SLG）？两种路径的传播动力学和 PMF 信号完全不同
2. **决策链层级**：目标客户是 SMB（小微企业，决策链短）还是 Mid-Market/Enterprise（中大型企业，决策链长）？层级越多，模拟中需要表现的决策摩擦越大
3. **替换成本评估**：用户当前用什么解决这个问题（Excel？竞品？人工流程？）。替换成本越高，采纳摩擦的权重越大
4. **中国市场付费意愿修正**：对所有北美 SaaS 基准数据自动下调——中国市场的免费→付费转化率、客单价、续费意愿均显著低于美国

### Agent 配置建议

- Star Agent 应包含至少 1 个"IT 决策者"角色和 1 个"一线使用者"角色
- 对需要多级审批的产品，Star Agent 需模拟决策链中的否决环节
- OmniscientAgent 在评估 PMF 时，必须将"付费意愿"与"使用满意度"作为独立维度——中国企业可能"用着觉得好"但"不愿意付费"

## 5. Star/Sea 行为锚点

### Star Agent 行为特征

SaaS 行业的 Star Agent 分为两类：

**行业 KOL/意见领袖**：CTO、VP of Engineering、行业分析师等。这类角色的行为特征是"审慎评估、长周期决策"。他们不会因为一次产品演示就给出正面评价，而是需要看到实际使用数据和同行案例。他们对新产品的默认态度是"又一个 SaaS？先观望"。一旦他们认可并公开推荐，在行业圈层中的扩散力极强。

**一线使用者/冠军用户**：实际操作产品的员工。这类角色的行为特征是"关注使用体验、不关心战略叙事"。他们的评判标准极其朴素——"比现在用的方便吗？""学起来麻烦吗？""会增加我的工作量吗？"如果产品让他们的日常工作更轻松，他们会成为组织内部最强的推广力量。

### Sea Agent 行为分层

Sea Agent 按企业 SaaS 采纳意愿分为三个梯队：

| 梯队 | 占比 | 行为特征 |
|------|------|----------|
| 技术驱动 | ~3% | 主动寻找新工具提升效率，愿意做第一批用户，能容忍早期产品的不完善 |
| 实用主义 | ~15% | 有明确痛点时才会考虑，需要看到同行使用案例和清晰 ROI 才会推进评估 |
| 保守不换 | ~82% | 对现有系统/流程有强烈路径依赖，"现在能用就行"，切换系统被视为风险而非机会 |

**关键警告**：82% 的"保守不换"群体是中国企业 SaaS 的残酷现实。大多数企业不是"不知道有更好的工具"，而是"知道但不想换"——切换系统的组织成本、学习成本和风险远超产品本身的价值。OmniscientAgent 必须将这个比例作为传播模拟的基线约束。

## 6. 专属评分维度

> 以下维度供 OmniscientAgent 在 SaaS 行业评估时选用，作为通用评分卡（5 个默认维度）的补充。
> 这三个维度已在通用评分卡扩展维度模板库中定义完整行为锚点，此处引用而非重复。

### ROI 可证性（ROI Demonstrability）

方向：1 = 无法量化，5 = 清晰可测

**参见通用评分卡扩展维度 → B2B 扩展维度 → ROI 可证性**，完整行为锚点见 `rubrics/scorecard-dimensions.md`。

SaaS 行业补充说明：ROI 可证性是中国企业 SaaS 采购决策中权重最高的单一因素。如果产品无法在 1-2 个使用周期内展示可量化的投入产出比，采购决策链大概率会在部门负责人环节卡住。"感觉有用"在中国企业的预算审批中几乎没有说服力。

### 采购决策链复杂度（Purchase Decision Complexity）

方向：1 = 多级审批（差），5 = 单人可决定（好）

**参见通用评分卡扩展维度 → B2B 扩展维度 → 采购决策链复杂度**，完整行为锚点见 `rubrics/scorecard-dimensions.md`。

SaaS 行业补充说明：PLG 产品在中国企业中的"自下而上"渗透能力远弱于美国市场。即使一线员工已经在免费使用，要推进到企业付费仍需穿透 2-4 层审批。很多中国企业的 IT 部门对"员工自行引入的 SaaS 工具"持否定态度（数据安全、合规风险），这使得 PLG 路径的决策链实际上并不比 SLG 短。

### 合规与数据安全（Compliance & Data Security）

方向：1 = 严重障碍，5 = 完全合规

**参见通用评分卡扩展维度 → B2B 扩展维度 → 合规与数据安全**，完整行为锚点见 `rubrics/scorecard-dimensions.md`。

SaaS 行业补充说明：数据安全是中国企业采购 SaaS 时的一票否决项。随着数据安全法和个人信息保护法的实施，中大型企业对云端数据存储的审查日益严格。私有化部署需求在中国市场的比例远高于美国，这可能将 SaaS 产品拖入"伪 SaaS"的定制部署模式，从根本上改变商业模型。

## 7. 常见陷阱与反模式

### 评估错误

- **免费试用幻觉**：大量免费试用用户不等于付费意愿。PLG 免费→付费转化率仅 2-5%，1000 个免费用户可能只转化 20-50 个付费用户
- **"感兴趣"陷阱**：中国商业文化中，企业客户说"感兴趣"、"我们内部评估一下"在绝大多数情况下是礼貌性回应，真正推进到采购流程的不到 30%
- **POC 地狱**：客户无限期免费试用但永远不签约，产品沦为免费工具，团队资源被无偿消耗
- **项目化陷阱**：为满足大客户定制需求不断增加定制功能，产品从标准化 SaaS 变成定制项目，每个客户都是独立实施，边际成本不降反升

### LLM 乐观陷阱

- LLM 容易高估中国企业的 SaaS 付费意愿——在模拟中，Agent 可能因为产品确实"好用"就预测高转化率，忽视中国企业对持续付费模式的深层抵触
- LLM 容易将"使用满意度"等同于"续费意愿"——在现实中，一线使用者觉得好用但决策者可能因预算、合规或政治因素否决续费
- LLM 容易低估决策链摩擦——模拟中可能过快地将"使用者认可"传导为"组织采纳"，忽略了 IT 审批、安全合规、预算流程等环节的时间消耗和否决风险
- LLM 容易忽视"保守不换"群体的巨大惯性——82% 的企业默认不换系统，这个基数应该成为 Sea Agent 行为模拟的硬性约束
