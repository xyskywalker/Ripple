# 应用商店/数字分发渠道画像

> 适用于通过应用商店和数字平台分发的产品场景：Apple App Store、Android 应用商店（华为应用市场、小米应用商店、OPPO 软件商店、应用宝）、微信小程序分发等。
> 传播逻辑为"榜单+评分驱动 → 推荐位曝光 → 下载转化 → 留存验证"，自然搜索排名和用户评分是核心增长飞轮。

## 渠道概述

应用商店是移动互联网时代最核心的数字分发渠道，也是用户获取成本（CAC）博弈最激烈的战场。中国市场的独特性在于 Android 生态的极度碎片化——不同于海外市场以 Google Play 为主，中国有华为、小米、OPPO、vivo、应用宝等数十个应用商店，每个商店有独立的审核规则、推荐算法和榜单机制。这意味着"上架"本身就是一项繁重的运维工作。

用户在应用商店的决策路径极短但筛选极其残酷：搜索/浏览 → 查看图标和标题（1-2 秒判断是否点入）→ 查看截图、评分和评论（5-10 秒决定是否下载）→ 下载。整个决策链中，评分低于 4.0 星的应用基本不会被下载，差评中的具体问题描述对潜在用户的劝退效应远超好评的吸引力。应用商店推荐位的生命周期通常只有 3-7 天，窗口期内如果没有积累足够的自然增长动能，曝光结束后下载量会断崖式下跌。

微信小程序作为"轻应用"分发渠道有其特殊性：获客门槛极低（无需下载安装），但留存挑战更大。小程序的次日留存率行业均值不到 15%，用户"用完即走"是常态而非例外。小程序的 PMF 验证需要特别关注"主动回访"行为——用户是否会在没有推送和入口引导的情况下，自主搜索或从收藏中重新打开小程序。

## 传播动力学

### 信息流转模式

应用商店的传播模型是"搜索 + 推荐 + 社交裂变"三引擎并行：

1. **搜索发现**：用户主动搜索关键词，应用在搜索结果中的排名决定曝光
2. **算法推荐**：商店根据下载量、评分、留存等指标推荐到编辑精选/榜单
3. **社交裂变**：用户通过微信/社交媒体分享应用给朋友（最强但最稀缺）
4. **评分飞轮**：高评分 → 更多推荐 → 更多下载 → 更多评分（正向/负向皆可）

### 基准参数

| 参数 | 数值 | 说明 |
|------|------|------|
| Wave 时间窗口 | 3-7 天 | 与商店推荐位周期和榜单更新频率一致 |
| 每轮衰减率 | 15-20% | 推荐位消失后下载量快速衰减 |
| 跨群体传播系数 | 0.30-0.40 | 数字分发天然无地理限制，跨群体传播能力强 |

### CES 行为权重

| 行为节点 | 权重 | 备注 |
|-----------|------|------|
| 搜索/浏览 | 1 | 被动曝光，几乎无信号价值 |
| 查看详情页 | 2 | 产生了初步兴趣 |
| 下载 | 4 | 采取行动，但下载成本为零（门槛极低） |
| 首次打开 | 5 | 超越了"下载即遗忘"的大多数 |
| D7 活跃 | 8 | 一周后仍在使用，强留存信号 |
| 评分/评论 | 7 | 主动表达意见，无论正负都是高参与信号 |
| 推荐好友 | 10 | 最强信号——愿意用社交信用为产品背书 |

## PMF 信号特征

### 强 PMF 信号

- **自然搜索下载量持续增长**——非买量/非推荐位驱动的有机增长
- **D30 留存率 >10%**——远超行业均值，说明产品解决了持续存在的需求
- **用户主动评分且均分 >4.5**——特别是带有具体使用场景描述的好评
- **卸载后重新安装**——用户离开后又回来，极强的需求验证信号
- **用户自发在社交媒体推荐**——不需要分享激励机制就愿意推荐
- **搜索品牌词的下载占比持续上升**——从品类搜索转向品牌搜索

### 弱 PMF 信号 / 噪音

- 买量带来的下载量激增（付费获客≠产品吸引力，停投即停增）
- 推荐位期间的下载高峰（3-7 天窗口期效应，不代表持续需求）
- 刷榜/ASO 优化带来的排名提升（虚假繁荣，不反映真实用户行为）
- 仅靠推送通知拉回的活跃用户（被动打开≠主动使用）
- 下载量大但 D1 留存极低（好奇心下载，打开一次就卸载）

## 关键指标锚点

| 指标 | 现实基准 | 说明 |
|------|----------|------|
| 商店浏览→查看详情 | 3-8% | 图标和标题的吸引力筛选 |
| 详情→下载转化 | 30-60% | 进入详情页后的下载率（评分是关键变量） |
| 下载→首次打开 | 60-80% | 部分用户下载后从未打开 |
| D1 留存率 | 20-35% | App Store 行业均值 |
| D7 留存率 | 10-20% | 一周后仍在使用 |
| D30 留存率 | 5-10% | 行业均值，>10% 已属优秀 |
| 微信小程序次日留存 | <15% | 小程序留存挑战更大 |
| 整体（曝光→D7活跃） | 1-3% | 漏斗全链路的残酷筛选 |
| 主动推荐率 | 0.3-1.5% | 愿意向朋友推荐应用的活跃用户比例 |

### 反乐观偏差锚点

- **App Store 平均下载 → D7 留存 <20%**——大多数下载的应用在一周内被遗忘
- **微信小程序次日留存 <15%**——"用完即走"是小程序的常态
- **低于 4.0 星基本不被下载**——评分是应用商店的生死线
- **90%+ 下载后 7 天内不再打开**——下载量是最具欺骗性的虚荣指标
- **刷榜/ASO 带来的下载 ≠ 真实用户**——这些下载不会产生留存和口碑
- **应用商店推荐位周期短（3-7 天）**——推荐结束后如果没有自然增长动力，下载量会断崖下跌
