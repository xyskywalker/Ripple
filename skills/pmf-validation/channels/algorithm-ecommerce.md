# 算法推荐电商渠道画像

> 代表平台：抖音电商、快手电商、拼多多推荐流
> 核心机制：算法匹配 → 兴趣激发 → 冲动决策

## 渠道概述

算法推荐电商是中国电商生态中增长最快的渠道形态。其核心逻辑是"货找人"——平台通过用户行为数据（浏览、停留、互动、购买历史）构建兴趣画像，将商品以短视频或直播形式推送到潜在消费者面前。用户并非带着明确购买意图进入，而是在内容消费过程中被激发出消费冲动。

这一机制决定了算法推荐电商的传播具有强脉冲特征：一条爆款视频可以在数小时内带来海量曝光，但衰减同样极为迅速。用户的决策窗口极短（通常3-15秒），决策链路从"看到内容"到"下单购买"可在几分钟内完成。这种冲动消费模式意味着高转化速度但低用户粘性——用户购买的是"内容激发的即时欲望"，而非经过深思熟虑的需求满足。

算法冷启动机制（抖音的ECPM竞价、快手的发现页推荐池）意味着新产品有机会获得初始曝光，但这种"算法给的流量"不等于真实市场需求。大量产品在算法推荐下获得了短暂的销量脉冲，却无法建立持续的复购行为，这是PMF验证中必须区分的关键信号。

## 传播动力学

### 信息流转模式

算法推荐电商的传播遵循"脉冲-衰减"模型：

1. **算法触发**：内容进入推荐池，获得初始曝光（通常200-500人的测试池）
2. **数据反馈**：完播率、互动率、转化率等指标决定是否进入更大流量池
3. **脉冲放大**：通过审核的内容在2-4小时内快速扩散，曝光量可呈指数增长
4. **急速衰减**：单条内容的生命周期极短，24小时后流量基本归零

### 基准参数

| 参数 | 参考值 | 说明 |
|------|--------|------|
| Wave 时间窗口 | 2-4h | 算法推荐周期短，单条内容生命力有限 |
| 每轮衰减率 | 20-25% | 衰减速度快，用户注意力被持续新内容替代 |
| 跨群体传播系数 | 0.30-0.40 | 算法推荐天然具备跨圈层分发能力 |

### CES 类行为权重参考

衡量用户参与度时，不同行为的信号强度差异显著：

- **完播/停留**（权重约2）：被动行为，仅表示内容不令人反感，不代表购买意向
- **点赞**（权重约1）：最轻量的正向反馈，信号极弱
- **评论**（权重约3）：表示内容引发了思考或情感反应，但需区分"看热闹"与"真实兴趣"
- **分享**（权重约5）：用户愿意以个人信用背书推荐给他人
- **下单购买**（权重约8）：最直接的付费行为信号
- **复购**（权重约10）：最强PMF信号，表示产品真正满足了持续需求

## PMF 信号特征

### 强PMF信号

- **自然流量转化**：在未投放付费广告的情况下，通过算法自然推荐产生持续订单，说明内容本身具备传播力且产品满足了真实需求
- **复购行为**：用户在首次冲动消费后主动回购，表明产品价值超越了"内容营造的即时欲望"
- **用户自发创作**：购买用户主动拍摄开箱、测评、使用体验类内容并发布，形成UGC裂变
- **评论区出现具体使用场景描述**：而非泛泛的"好用""不错"

### 弱PMF信号（警惕误判）

- 投流带来的高销量：付费流量下的转化不代表产品有自然需求
- 高曝光低转化：算法给了流量但用户不买单，说明产品与受众不匹配
- 单次爆款后沉寂：一条视频火了但后续内容无法复制效果
- 低客单价冲动订单+高退货率：用户"试试看"心态而非真实需求

## 关键指标锚点

以下为真实市场中的基线水平，OmniscientAgent在模拟中应以此为锚，避免过度乐观：

| 指标 | 基线范围 | 反乐观锚点 |
|------|----------|------------|
| 注意力捕获率（Attention） | 8-15% | 大部分推荐流商品被划走，平均停留不足3秒 |
| 行动转化率（Action） | 2-5% | 转化率1-3%已属优秀水平，多数商品低于1% |
| 推荐传播率（Recommend） | 0.5-2% | 用户主动分享商品的意愿极低，除非产品有极强的社交货币属性 |
| 退货率 | 20-40% | 冲动消费的必然结果，服饰类可达40%以上 |
| 自然流量占比 | <30% | 多数商家依赖付费投流，自然流量占比低 |
| 内容生命周期 | 24-48h | 算法推荐内容的有效曝光窗口极短 |

**核心反乐观认知**：算法冷启动给予的初始流量是"平台测试"而非"市场验证"。大量产品在推荐流中获得了短暂曝光却无法沉淀为持续销售，将算法分配的流量误读为PMF信号是最常见的判断错误。真正的PMF验证需要观察算法推荐停止后，产品是否仍有自然搜索和复购流量。
