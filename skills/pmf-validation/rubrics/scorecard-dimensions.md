# PMF 评分卡维度定义与行为锚点

> 所有维度统一方向：1 = 最差，5 = 最好。评分为整数，不允许半分。
> 每个评分必须对应下方行为锚点描述，不得自由发挥。

## 默认维度（5 个，所有 PMF 评估必须包含）

### 1. 需求共鸣度（Demand Resonance）

方向：1 = 无共鸣，5 = 强共鸣

| 分数 | 行为锚点 |
|------|----------|
| 1 | 用户听到产品描述后无反应，不追问、不关注。产品解决的问题与用户无关，或用户不认为这是一个问题。 |
| 2 | 少数用户表示"有点意思"但不会主动了解。产品触及的痛点存在但优先级很低，用户有更紧迫的需求。 |
| 3 | 部分用户（约 20-30%）表现出明确兴趣，愿意试用。产品解决了一个真实但非核心的痛点，或核心痛点的解决方案不够彻底。 |
| 4 | 多数目标用户表现出强兴趣，主动追问细节和使用方式。产品精准命中了一个未被良好满足的需求。 |
| 5 | 用户表示"这正是我需要的"，会主动推荐给他人。产品解决了用户长期忍受但找不到好方案的核心痛点。 |

### 2. 传播潜力（Propagation Potential）

方向：1 = 无传播力，5 = 强传播力

| 分数 | 行为锚点 |
|------|----------|
| 1 | 无人愿意传播或讨论。产品信息止于接触者，不产生任何二次传播。 |
| 2 | 仅在被直接询问时才会提及，不会主动分享。传播需要外力推动（如广告投放）。 |
| 3 | 部分用户会在相关场景中推荐（如被问"有什么好用的XXX"时），但不会主动发起话题。 |
| 4 | 用户愿意主动分享使用体验，在社交场景中作为"话题"提及。产品有一定的社交货币价值。 |
| 5 | 用户自发创作内容并主动分享（UGC），产品本身成为传播载体。"你一定要试试这个"成为用户间的日常推荐语。 |

### 3. 竞争差异化（Competitive Differentiation）

方向：1 = 无差异化，5 = 强差异化

| 分数 | 行为锚点 |
|------|----------|
| 1 | 与现有产品无法区分，用户认为"已经有很多类似的了"。在拥挤的市场中没有辨识度。 |
| 2 | 有细微差异但用户难以感知或不在意。差异化停留在功能层面，未转化为用户价值。 |
| 3 | 在某个维度有明显差异，但用户在选择时仍会犹豫。差异化存在但不足以成为决定性因素。 |
| 4 | 用户能清晰说出"这个产品不一样在于..."，差异化是选择的重要理由之一。 |
| 5 | 用户认为没有替代品，"只有这个能做到XXX"。产品在目标场景中占据不可替代的位置。 |

### 4. 采纳摩擦（Adoption Friction）

方向：1 = 高摩擦（差），5 = 低摩擦（好）

| 分数 | 行为锚点 |
|------|----------|
| 1 | 需要大量学习、迁移数据、建立信任、或组织审批。用户即使想用也因门槛过高而放弃。 |
| 2 | 需要一定的学习成本或配置，但有耐心的用户可以完成。部分用户因嫌麻烦而流失。 |
| 3 | 上手需要 5-15 分钟的引导，但之后可以正常使用。存在一些摩擦点但不致命。 |
| 4 | 注册/开通流程简单，核心功能 5 分钟内可体验。轻微的使用门槛不构成障碍。 |
| 5 | 即开即用，零切换成本。用户可以无缝从现有方案迁移，或不需要替代任何现有工具。 |

### 5. 持续价值（Sustained Value）

方向：1 = 一次性（差），5 = 持续价值（好）

| 分数 | 行为锚点 |
|------|----------|
| 1 | 用户尝试后不会再用。产品提供的是一次性的新鲜感，没有复访理由。 |
| 2 | 偶尔会用但不是常规工具。用户记得有这个产品但使用频率很低。 |
| 3 | 定期使用但可有可无。如果产品消失，用户会觉得不便但能找到替代方案。 |
| 4 | 已成为工作/生活流程的一部分，定期使用并有一定依赖。产品消失会造成明显不便。 |
| 5 | 用户产生依赖，"离不开"。产品深度嵌入用户流程，替换成本极高。停止使用会严重影响效率或体验。 |

## 扩展维度模板库

> OmniscientAgent 可根据产品领域从以下模板库中选取 0-3 个扩展维度。
> 选取的维度具有完整行为锚点，不允许自由发明新维度。

### B2B 扩展维度

#### ROI 可证性（ROI Demonstrability）

方向：1 = 无法量化，5 = 清晰可测

| 分数 | 行为锚点 |
|------|----------|
| 1 | 无法量化 ROI，产品价值停留在"感觉不错"层面。采购决策者找不到数据支撑。 |
| 2 | 可以定性描述价值但难以量化。"应该有用但说不清楚有多大用。" |
| 3 | 可以通过间接指标估算 ROI，但需要较长的观察期。 |
| 4 | 产品自带数据看板或报告，ROI 可在一个使用周期内可见。 |
| 5 | ROI 清晰可测，可直接关联到收入增长/成本降低。"用了之后 XX 指标提升了 YY%"。 |

#### 采购决策链复杂度（Purchase Decision Complexity）

方向：1 = 多级审批（差），5 = 单人可决定（好）

| 分数 | 行为锚点 |
|------|----------|
| 1 | 需多级审批、招投标、合规审查。采购周期以月甚至季度计。 |
| 2 | 需要部门主管和 IT 审批。采购周期数周。 |
| 3 | 需要直属上级审批，但流程相对简单。 |
| 4 | 团队负责人可在预算范围内自主决定。 |
| 5 | 个人用户可直接付费使用，无需任何审批。 |

#### 合规与数据安全（Compliance & Data Security）

方向：1 = 严重障碍，5 = 完全合规

| 分数 | 行为锚点 |
|------|----------|
| 1 | 有严重合规障碍（如数据出境、行业准入限制），无法通过安全审查。 |
| 2 | 存在合规问题但理论上可解决，需要额外的安全措施和审查流程。 |
| 3 | 基本合规但需要补充某些认证或安全措施。 |
| 4 | 满足大部分行业合规要求，仅需少量适配。 |
| 5 | 完全合规，已通过主流安全认证（ISO 27001、SOC 2 等）。 |

### 消费品扩展维度

#### 复购意愿（Repurchase Intent）

方向：1 = 不会再买，5 = 定期复购

| 分数 | 行为锚点 |
|------|----------|
| 1 | 不会再买，一次体验即可满足好奇心。 |
| 2 | 如果打折/有优惠券可能会再买，但不会主动购买。 |
| 3 | 会在有需求时考虑再买，但不是首选。 |
| 4 | 已加入常备清单，会定期复购。 |
| 5 | 会定期复购且会囤货。断货时会焦虑寻找替代。 |

#### 价格敏感度（Price Sensitivity）

方向：1 = 价格是首要障碍，5 = 价格不是考虑因素

| 分数 | 行为锚点 |
|------|----------|
| 1 | 价格是首要障碍，多数目标用户认为"太贵了"。产品价值感不足以支撑定价。 |
| 2 | 价格引发犹豫，用户会反复比价后才决定。"想买但要等打折。" |
| 3 | 价格在可接受范围内但不算便宜，用户需要说服自己"值得买"。 |
| 4 | 用户认为定价合理，价格不是决策的阻碍因素。 |
| 5 | 价格不是考虑因素，用户认为"这个价格很划算"或"贵一点也无所谓"。 |

#### 社交货币价值（Social Currency Value）

方向：1 = 不会晒，5 = 必须晒

| 分数 | 行为锚点 |
|------|----------|
| 1 | 不会在社交场景中提及或分享，产品无社交属性。 |
| 2 | 被问到时可能提及但不会主动分享。 |
| 3 | 在合适的场景下会分享（如产品确实解决了某个问题），但不常见。 |
| 4 | 会主动在社交媒体晒出或向朋友推荐，产品使用本身有社交价值。 |
| 5 | 必须晒朋友圈/社交媒体，产品是身份/品味/生活方式的表达。不分享就浪费了。 |
